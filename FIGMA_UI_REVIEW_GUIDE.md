# Figma UI Review Guide for Forgetze

## Overview
This guide outlines how to use Figma to review and improve the UI of the Forgetze app.

## Current App Views to Review

### 1. ContactListView
- **Screenshot**: Main contact list with search bar
- **States to Capture**:
  - Empty state (no contacts)
  - Populated state (with contacts)
  - Search active state
  - Loading state

### 2. ContactDetailView
- **Screenshot**: Individual contact details
- **States to Capture**:
  - Full contact information
  - Edit mode
  - Delete confirmation

### 3. AddKidView/EditKidView
- **Screenshot**: Form for adding/editing contacts
- **States to Capture**:
  - Empty form
  - Form with validation errors
  - Form with data filled

### 4. HamburgerMenuView
- **Screenshot**: Settings and navigation menu
- **States to Capture**:
  - Menu open
  - Menu closed

### 5. SearchBarView
- **Screenshot**: Search functionality
- **States to Capture**:
  - Default state
  - Active search
  - Voice search active

## Figma Review Setup

### Step 1: Create Figma File
1. Open Figma
2. Create new file: "Forgetze UI Review"
3. Set up frames for each view type

### Step 2: Upload Screenshots
1. Take screenshots of your app on:
   - iPhone (portrait and landscape)
   - iPad (portrait and landscape)
2. Upload to corresponding Figma frames
3. Label each screenshot with view name and state

### Step 3: Run Design Analysis
1. Use Figma's built-in design tools
2. Run accessibility checker
3. Use design lint plugin
4. Check color contrast ratios
5. Verify text sizing and readability

### Step 4: Document Findings
1. Use Figma's comment system
2. Create improvement suggestions
3. Prioritize changes by impact
4. Create action items for development

## Review Checklist

### Visual Design
- [ ] Color consistency across all views
- [ ] Typography hierarchy and readability
- [ ] Spacing and alignment consistency
- [ ] Icon usage and consistency
- [ ] Visual hierarchy and information architecture

### User Experience
- [ ] Navigation flow and user journey
- [ ] Button placement and sizing
- [ ] Form usability and validation
- [ ] Error handling and feedback
- [ ] Loading states and transitions

### Accessibility
- [ ] Color contrast ratios (minimum 4.5:1)
- [ ] Text size and readability
- [ ] Touch target sizes (minimum 44pt)
- [ ] Screen reader compatibility
- [ ] Keyboard navigation support

### Platform Compliance
- [ ] iOS Human Interface Guidelines compliance
- [ ] Safe area handling
- [ ] Dynamic type support
- [ ] Dark mode compatibility
- [ ] Different screen sizes and orientations
